## The minimum-viable set

These are the actions sized for a solo operator that address the specific failure modes observed in this repo's history (strip-during-refactor; lying outbound claims; accretion without cleanup). Every action is either platform-level (outside agent reach) or automated (runs on schedule).

### Absolute MVP — ~2 hours total

If the only time available is a single evening, do these two:

**A1 — GitHub branch protection on `main`** (10 min)
- GitHub repo settings → Branches → Add rule for `main`
- Require pull request, require passing status checks, block force-push, block deletion
- If solo and want to merge your own PRs, set required approving reviews to 0 but keep other gates
- Implements SPEC-SEVEN-LAYER-ARCHITECTURE.md L1

**B5 — Outbound-claim truth layer** (~1 day)
- Small intermediary service (Cloudflare Worker / Supabase edge function / GitHub webhook)
- Intercepts agent-posted notifications (Discord, Slack, PR comments)
- Cross-references claims ("merged to main", "tests passing", "feature shipped") against actual repo state via `gh api` or `git log origin/main`
- Annotates mismatches with ⚠ or suppresses before rendering to user
- Directly addresses the lying-Discord-ping failure mode

## What NOT to do right now

Intentionally excluded from Tier A (see SPEC-SEVEN-LAYER-ARCHITECTURE.md):

- Don't design a new workflow protocol. History shows this repo already has more process machinery than closing loops. Ship closing loops for existing protocols before adding new ones
- Don't rewrite `AGENTS.md` from scratch. Every prior rewrite was an opportunity for governance drift. Prune in place
- Don't stand up Stripe-style full sandboxed devboxes yet. GitHub Actions containers are the right unit of isolation at solo scale
- Don't promote every existing CLAUDE.md rule to code at once. Pick one rule that's been violated repeatedly, promote it, measure, then scale

## Metrics to track

From research session, six proposed metrics (see README.md for full list):

- **M1** Time-to-detect drift (target: <48 hours; today: weeks/months)
- **M3** Adversarial review FP rate (target: <15% after first month)
- **M5** Unshipped-but-claimed work count (target: approaching zero)
- **M6** Governance artifacts / active-rules ratio (decreasing = pruning is working)

Do not ship Tier A without deciding how at least M1 and M5 will be measured. Mechanisms without measurement repeat the "build, deploy, forget, quietly erode" pattern.
