Agentic Runbooks vs. Auto-Researcher

Exploring how these two systems interact

Agentic Runbooks (.specs/agentic-runbooks.md)
Primary Goal

Interactive, human-in-the-loop design & implementation env.

  • Declarative manifests for code changes
  • Durable workflows (Effect-TS)
  • Integration with Supabase & MCP
  • Human-centric: "Help me build this feature"
Auto-Researcher (@autoresearch-agents)
Primary Goal

Autonomous optimization of agents via population search.

  • MAP-Elites, ADAS, Go-Explore algorithms
  • Parallel eval workers (Cloud Run)
  • Distributed archive (Firestore/GCS)
  • Autonomous-centric: "Find the best agent while I sleep"

The Interaction Opportunity

The Agentic Runbook is the workspace, and the Auto-Researcher is a specialized inhabitant or engine that can be hosted within it.
