### 2/6 -- Problem

The bottleneck on AI agents isn't intelligence. It's memory.

Ask Claude to research something complex. After ~15 minutes, it starts re-deriving conclusions it already reached. It forgets what it tried. It can't build on its own earlier thinking.

Context windows don't solve this. Structure does.

---

### 3/6 -- Proof

We built Thoughtbox -- an MCP server that gives agents structured cognition.

Not "save a note." Real cognitive operations:
- Belief snapshots (track how understanding evolves)
- Decision frames (weigh options with evidence)
- Knowledge graphs (connect ideas with typed relations)
- Progress checkpoints (know what's done, what's next)

---

### 4/6 -- Insight

The research session found something we didn't expect.

The agent didn't just collect information. It revised its own beliefs 12 times. It found connections between papers that weren't cited together. It built a Five Pillars framework that synthesized 30+ paradigms into a coherent architecture.

This is what happens when you give an agent room to think.

---

### 5/6 -- How it works

Thoughtbox runs as an MCP server. Connect it to Claude Code with one JSON config.

The agent gets tools like `thoughtbox_execute` and `thoughtbox_search`. It writes typed thoughts, queries its own prior reasoning, and builds knowledge graphs -- all persisted across sessions.

Open source MCP server. Hosted version at thoughtbox.dev.
