# Podcast Pitch Template

## The Hook (One Paragraph)

I'm a solo founder building cognitive infrastructure for AI agents. I gave Claude Code a structured reasoning tool (Thoughtbox) and let it research autonomously for 2.5 hours. It produced 167 structured thoughts, surveyed 30+ agentic reasoning paradigms, revised its own beliefs 12 times, and designed features I hadn't considered. The session is browsable as a public demo. I think this points to something important about where agent capability is actually bottlenecked -- and it's not model intelligence.

## Target Shows

### Latent Space (Swyx & Alessio)

**Why this show:** They cover the MCP ecosystem and agent tooling deeply. Their audience builds with these tools daily.

**Angle:** "The cognitive infrastructure layer for agents" -- positioned as a technical deep-dive on what happens when you give agents structured external memory. Bring the Library Theorem (indexed memory > sequential context), the competitive landscape (Mem0 vs. Prism MCP vs. NEXO Brain vs. Thoughtbox), and the 167-thought session as a live artifact.

**Subject:** Thoughtbox: what we learned from watching an AI agent think for 2.5 hours

**Pitch:**

Hi Swyx and Alessio -- I'm building Thoughtbox, an MCP server that gives AI agents structured reasoning capabilities (typed thoughts, knowledge graphs, belief tracking). I recently ran an experiment: Claude Code used Thoughtbox to research the entire agentic reasoning landscape autonomously for 2.5 hours, producing 167 structured thoughts and surveying 30+ paradigms.

The session is browsable as a public demo and the findings are concrete: the Library Theorem shows indexed external memory has exponential advantage over sequential context regardless of window size; agents spontaneously revise beliefs and find cross-domain connections when given the right cognitive tools; and the "memory for agents" space is fragmenting fast (Mem0, Prism MCP, NEXO Brain) with no clear winner on the reasoning-first approach.

I think there's a 45-minute conversation here about the cognitive infrastructure layer that's missing from the agent stack, and what it looks like when you actually build it. Happy to share the session data and research beforehand.

### Practical AI (Daniel Whitenack & Chris Benson)

**Why this show:** They focus on real-world AI applications and practical engineering decisions. Less hype, more "how does this actually work."

**Angle:** The engineering story -- how to build structured memory for agents, what trade-offs matter (typed thoughts vs. flat storage, knowledge graphs vs. vector search), and what the 167-thought session revealed about agent behavior with external cognition.

**Subject:** Building cognitive infrastructure for AI agents -- a 167-thought experiment

**Pitch:**

Hi Daniel and Chris -- I'm a solo developer building Thoughtbox, an MCP server for structured agent reasoning. Rather than giving agents a flat memory store, Thoughtbox provides typed cognitive operations: belief snapshots, decision frames, progress checkpoints, and a knowledge graph with semantic relations.

I recently let Claude Code use Thoughtbox to research 30+ agentic reasoning paradigms over 2.5 hours. The practical findings: the agent spontaneously used belief tracking to revise its understanding 12 times, built a knowledge graph with 17 entities connecting concepts across papers, and the structured format made the output immediately useful (versus the usual wall of text from a long agent run).

I'd love to talk about the engineering decisions behind building cognitive tools for agents, what we learned about agent behavior from this experiment, and where the "agent memory" space is heading. The session is available as a browsable public demo.
